  The Research Paper of ForkBase is Selected by The Morning Paper
Date: 1 JUNE 2018   |   Category: Technology   |   Source: The Morning Paper (by Adrian Colyer)

ForkBase is a data storage system designed to support applications that need a combination of data versioning, forking, and tamper proofing. The prime example being blockchain systems, but this could also include collaborative applications such as GoogleDocs. Today for example Ethereum and HyperLedger build their data structures directly on top of a key-value store. ForkBase seeks to push these properties down into the storage layer instead.

ForkBase is a system that supports high-level properties demanded in many modern applications, including data versioning, collaboration and security. Hyperledger++

Hyperledger++ is an enhanced blockchain infrastructure of Hyperledger. It uses trusted hardware to improve trust relation between nodes and its throughput by a wide margin. The design of Hyperledger++ follows two novel but critical principles, namely scale up by scaling down and prioritize consensus messages, that enable the blockchain consensus protocols to scale.

  Storage Support for Multi-tenant Healthcare Analytics

Data is the base of any kind of analytics. Healthcare data such as EHR can usually be used for multiple purposes of analytics, which form the collaboration of healthcare analytics among doctors, medical researchers, health agencies and institutes, etc. Collaborative analytics requires different formats of the data produced by intricate processing such as cleaning, transformation and integration. This essentially requires database to store versions of the data, which not only introduces massive data redundancy as each data version is stored as a stand-alone copy, but also renders data versions hard to govern.

ForkBase is a collaboration-centric storage system that fully supports collaborative analytics. Unlike traditional database being data-centric only, ForkBase facilitates data forking, version management and access control to the end of multi-tenant analytics. Moreover, ForkBase is endowed with a novel data de-duplication technology that can remarkably reduce data redundancy among data versions in the physical storage. Technical details are explained in our research paper. MediLOT Technologies provide consultation for the deployment of ForkBase and technical support for client application integration.
